They tend to "strip out" any injected form elements, leaving only the main form. In addition, USWDS handles modals by first destroying the element, then repopulating it toward the end of the page. In effect, this means that the modal is not, and cannot, be surrounded by any form element at compile time. The current workaround for this is to use javascript to inject a hidden input, and bind submit of that element to the click of the confirmation button within this modal. This is controlled by the class `dja-form-placeholder` on the button. In addition, the modal element MUST be placed low in the DOM.
